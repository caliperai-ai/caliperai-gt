# Handling Rejections

## Why Tasks Get Rejected

Tasks are rejected when quality doesn't meet standards. Common reasons:

### Annotation Issues
- Missing objects
- Wrong classifications
- Inaccurate box positions
- Poor tracking

### Quality Issues
- Boxes floating above ground
- Oversized boxes
- Missing attributes
- Inconsistent dimensions

### Completeness Issues
- Not all frames annotated
- Tracks with gaps
- Missing required classes

## When You Receive a Rejection

1. **Read feedback carefully** - Note specific issues mentioned
2. **Review flagged items** - Look at problem annotations
3. **Understand the issue** - Make sure you know what's wrong
4. **Ask questions** - If unclear, message the reviewer

## Fixing Rejected Work

### Step by Step
1. Open the rejected task
2. Review QA feedback/comments
3. Navigate to problem areas
4. Make corrections
5. Run AI QA again
6. Verify fixes resolved issues
7. Resubmit

### Finding Issues
- Use **AI QA panel** to see flagged items
- Click on issues to jump to them
- Sort by severity (critical first)

## Preventing Rejections

1. **Self-review** before submitting
2. **Run AI QA** and fix issues
3. **Follow guidelines** exactly
4. **Check edge cases** carefully
5. **Verify tracks** play smoothly
6. **Complete all frames**

## Learning from Rejections

- Each rejection is a learning opportunity
- Note common mistakes
- Review project guidelines
- Ask for clarification if needed
- Improve over time

## Communication

If you disagree with rejection:
- Add comment explaining your reasoning
- Provide evidence from guidelines
- Discuss with reviewer
- Escalate if unresolved
