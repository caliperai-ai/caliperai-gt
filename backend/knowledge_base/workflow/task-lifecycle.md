# Task Lifecycle

## What is a Task?

A **task** is a unit of work assigned to an annotator. It contains:
- One or more **scenes** to annotate
- A specific **taxonomy** (classes and attributes)
- Status tracking through the workflow

## Task Stages

### 1. Annotation Stage
- **Assigned**: Task created and assigned to you
- **In Progress**: You're actively working on it
- **Submitted**: You've finished and submitted for review

### 2. QA Stage
- QA reviewer checks your work
- **Accepted**: Quality meets standards
- **Rejected**: Needs corrections (returns to you)

### 3. Customer QA Stage (if applicable)
- Customer reviews annotations
- Final acceptance or rejection

### 4. Accepted
- Task complete and finalized

## Working on a Task

### Finding Your Tasks
1. Go to **My Tasks** page
2. Filter by status (Assigned, In Progress)
3. Click to open task

### Starting Work
1. Open the task
2. Click **Start Work** or **Resume**
3. Task status changes to "In Progress"

### Saving Progress
- **Auto-save**: Changes save automatically
- **Manual save**: Press `Ctrl+S`
- Progress is preserved if you close browser

### Submitting
1. Complete all required annotations
2. Verify quality (check AI QA results)
3. Click **Submit for Review**
4. Task moves to QA stage

## Task Information

Each task shows:
- **Scene count**: How many scenes to annotate
- **Frame count**: Total frames across scenes
- **Progress**: Completed vs total frames
- **Due date**: When task is expected complete
- **Priority**: Urgency level

## Tips

1. **Check QA** before submitting
2. **Complete all frames** in assigned scenes
3. **Follow taxonomy** exactly as specified
4. **Note blockers** if you can't complete
5. **Ask questions** if requirements unclear
