# Glossary of Terms

## A

**Annotation**: A label or marking applied to data to identify objects or regions.

**Attribute**: Additional properties of an annotation (e.g., occluded, truncated).

**Auto-save**: Automatic saving of work at regular intervals.

## B

**BEV (Bird's Eye View)**: Top-down view of the scene, looking down the Z-axis.

**Bounding Box**: A rectangular shape (2D) or cuboid (3D) that encloses an object.

## C

**Calibration**: The alignment data between sensors (camera to LiDAR).

**Class**: The category or type of an annotated object (e.g., car, pedestrian).

**Cuboid**: A 3D rectangular box used to enclose objects in LiDAR data.

## D

**Dataset**: A collection of scenes organized together for a project.

## E

**Ego Vehicle**: The vehicle on which sensors are mounted (the "self" vehicle).

## F

**Frame**: A single point in time; one capture from all sensors.

**Fusion**: Combining data from multiple sensors (LiDAR + camera).

## G

**Ground Plane**: The estimated surface level where the ground is.

**Ground Truth**: The correct/reference annotations used to train AI.

## I

**Interpolation**: Automatically filling in frames between manually annotated keyframes.

**IoU (Intersection over Union)**: A measure of overlap between two regions.

## K

**Keyframe**: A manually annotated frame used as reference for interpolation.

## L

**LiDAR**: Light Detection and Ranging; sensor that creates 3D point clouds.

## O

**Occlusion**: When an object is partially hidden by another object.

**Origin**: The center point of the coordinate system (0, 0, 0); typically the LiDAR location.

## P

**Point Cloud**: A set of 3D points representing surfaces detected by LiDAR.

**Propagation**: Automatically extending a track to additional frames.

## Q

**QA (Quality Assurance)**: The review process to ensure annotation quality.

## R

**RAG**: Retrieval-Augmented Generation; AI technique for informed responses.

## S

**SAM**: Segment Anything Model; AI for automatic image segmentation.

**Scene**: A sequence of frames from a single recording/drive.

**Semantic**: Relating to meaning (e.g., semantic placement = logical location).

## T

**Taxonomy**: The defined set of classes and attributes for a project.

**Track**: A sequence of annotations representing the same object across frames.

**Truncation**: When an object extends beyond the camera view boundary.

## Y

**Yaw**: Rotation around the vertical (Z) axis; heading direction.
